모든 파일 스테이징 - git add .
여러 파일 스테이징 - git add 파일명1 파일명2
상태창 - git status
커밋 내역 조회 - git log --all --oneline
커밋 그래프로 보기 - git log --all --graph

커밋 - git commit -m "커밋 메시지"
전 버전과 차이점을 보여준다 - git difftool

프로젝트 복사본 , 커밋의 복사본 - git branch 브랜치이름
브랜치 이동 - git switch 브랜치이름
브랜치 삭제 - git branch -d 브랜치이름
브랜치 합치기 - git merge 브랜치이름
#서브브랜치에서 개발한 것은 서브브랜치에서만 가능
#메인 브랜치로 스위치하면 그 파일들은 없어짐
#브랜치를 만들어서 작업하고, 작업이 완료되면 메인 브랜치로 합치는 방식으로 진행
#같은 파일 같은 줄을 수정하면 merge로 합칠때 충돌이 발생함

merge 완료된 브랜치 삭제 - git branch -d 브랜치이름
merge 안한 브랜치 삭제 - git branch -D 브랜치이름
git rebase 브랜치이름 - 브랜치의 신규시작점을 최신으로 옮길 수 있다.
#rebase쓰는 이유 : 브랜치 없이도 코드 잘짜는 고수 같음
#3-way merge는 너무 복잡해진다. 간단하게 하려면 rebase를 쓰는게 좋다.
#rebase 단점 : conflict엔딩이 많이 남
깔끔한 merge - git squash 브랜치이름

git은 버전관리 목적
파일 복구 가능, 커밋 복구 가능, 과거로 이동가능
파일 복구 하는법 : git restore 파일명
특정 commit 시점으로 파일 복구하는법 : git restore --source 커밋해시 파일명
스테이징된 파일을 unstaged로 변경(파일단위 복구) : git restore --staged 파일명 
commit 취소 하는 법 : git revert 커밋해시
commit 여러개취소가능 : git revert 커밋해시1 커밋해시2
최근 commit 취소가능 : git revert HEAD
과거로 모든걸 되돌리기 : git reset --hard 커밋해시
리셋인데 변동사항 지우지말고 스테이징 해놓기 : git reset --soft 커밋해시
리셋인데 변동사항 지우지말고 unstaged로 해놓기 : git reset --mixed 커밋해시
